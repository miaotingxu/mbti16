# MBTI 性格测试网站 - 产品需求文档 (PRD)

## 1. 项目背景
MBTI (Myers-Briggs Type Indicator) 是一种广泛使用的性格测试工具。本项目旨在开发一个简洁、美观且易用的 MBTI 在线测试网站，帮助用户了解自己的性格类型（16型人格），并提供详细的性格分析和职业建议。

## 2. 功能需求

### 2.1 首页 (Landing Page)
- **Banner区域**：
  - 吸引人的标题（如“探索真实的自己”）。
  - 简短的介绍文案。
  - 醒目的“开始测试”按钮。
- **功能介绍**：
  - 简单介绍 MBTI 的四个维度（E/I, S/N, T/F, J/P）。
  - 展示 16 种人格类型的概览（图标或卡片）。
- **页脚**：
  - 版权信息。
  - 隐私政策链接。

### 2.2 测试页 (Quiz Page)
- **进度条**：顶部显示当前答题进度（例如：第 5/28 题）。
- **题目卡片**：
  - 显示当前题目内容。
  - 提供两个互斥的选项（A/B）。
  - 切换动画：选择后自动平滑过渡到下一题。
- **控制按钮**：
  - “上一题”按钮（允许修改答案）。
  - “提交”按钮（仅在最后一题出现）。
- **防误触**：未选择选项时无法进入下一题。

### 2.3 结果页 (Result Page)
- **性格类型展示**：
  - 显示测试结果代码（如 INTJ）。
  - 显示性格称号（如“建筑师”）。
  - 对应的插画或图标。
- **维度分析**：
  - 使用进度条或雷达图展示四个维度的倾向百分比（例如：内向 80% vs 外向 20%）。
- **详细解读**：
  - **性格特征**：详细的文本描述。
  - **优势与劣势**：列表展示。
  - **适合职业**：推荐的职业领域。
  - **标签**：关键词标签（如“完美主义”、“逻辑强”）。
- **分享与重测**：
  - “分享结果”按钮（生成图片或复制链接）。
  - “重新测试”按钮。

## 3. 用户流程图 (User Flow)

1. **进入首页** -> 点击“开始测试”
2. **进入测试页** -> 依次回答 28 道题目 -> 自动跳转或点击提交
3. **计算中** -> (可选：显示简短的加载动画)
4. **进入结果页** -> 查看详细报告 -> 分享或重测

## 4. UI/UX 建议
- **设计风格**：现代、极简、清新。建议使用柔和的背景色搭配鲜明的强调色。
- **响应式设计**：完美适配移动端（手机）和桌面端。
- **交互体验**：
  - 答题时的卡片切换要有流畅的过渡动画。
  - 按钮要有清晰的点击反馈。
- **字体**：清晰易读的无衬线字体。

## 5. 数据需求
- 需要准备 28 道标准的 MBTI 测试题。
- 需要准备 16 种人格类型的完整数据库（包含描述、优缺点、职业等）。
